Access Control
 Previous Top Next

The Access Control Dialog is used to set the access rights for the fields contained within an input model. The access control and rights of an user regarding a specific record are enforced using the input model on which the record is based.

The access control rights can be set for the user categories or for individual users by using the form under the relevant tab. Both the tab used for setting access rights for the user categories and the tab for setting individual user rights have the same functionality.

When the access rights are enforced, the system first checks whether the user has specific rights for a field. If no rights are found, the access rights of the user category to which this user belongs are enforced. The title bar of the Access Rights Dialog shows the name of the field for which the access rights are being set. You can select a user category or specific user from the list on the left-hand side of the dialog.

For each entity (user or user category) you can set the following rights:

·    Read

Gives the entity read access to the field. This means that this field will be visible to the entity when a record based on this input model is viewed in the Termbrowser.

·    Update

Gives the entity the ability to update this field. The entity may then create or edit this field when creating/editing a record based on this input model.

·    Delete

Gives the entity the ability to delete this field when editing a record based on this input model.

·    Export

Gives the entity the ability to export this field when exporting a list of terms.

Once the access rights have been set and you are satisfied with the access control for a specific input model, click on the Save button to save the new access rights or on the Cancel button to discard any unsaved changes made.

Remember:  Access control can only be set on an input model that has already been saved.

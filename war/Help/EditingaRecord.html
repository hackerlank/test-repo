Editing a Record
 Previous Top Next

Signed-on users with the relevant rights can edit an existing record by clicking on the Edit Record button. If a guest user attempts to edit records, the link will be disabled.

Once you have clicked on the Edit Record button, the fields that are specified in the records input model will be displayed. Any data that has already been entered as part of the record will be displayed in the respective fields. You can add new fields and edit or remove existing fields.

Adding a field

To add a new, empty field to the record, click on the green plus sign next to a field.

This will display a dialog, from which you can select the field to be added.

The fields you will be able to add will be determined by your access rights and whether you clicked the icon next to a record field or index field. If the icon was clicked next to a record field, then all the unpopulated record fields and index fields contained in the input model will be displayed and can be added. If the icon was clicked next to an index field, then all the attribute fields associated with that index field will be displayed and can be added.

Removing a field

A field can be removed by clicking on the red cross next to the field.

This will remove the data in the field from the record. Only users with the relevant rights may remove fields.

The selected source and target languages determine the order in which the index fields are displayed. The source language index will be displayed first, followed by the target language index field, if one was specified. The remaining index fields will be displayed in the order that was defined in the input model.

Once you are satisfied with the edited record and have completed all of the mandatory fields, click on the Save button in the Control Box to save the changes made to the record. The edited term will be displayed in the Termbrowser.
